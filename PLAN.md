## 📁 Git-workflow без конфликтов

| Зона | Кто трогает |
|---|---|
| `src/server/**` | Dev A |
| `src/client/**` | Dev B |
| `src/gui/**` | Dev B |
| `src/shared/**` | Оба (по согласованию — обычно один пишет, другой ревьюит PR) |
| `assets/**` (модели .rbxm) | Dev A (бэкап сцены) |
| `CLAUDE.md`, `PLAN.md`, `README.md` | Оба, по необходимости |
| Сцена в Studio (карта, спавны) | Dev A через Team Create |
| GUI в StarterGui (через Studio) | Dev B через Team Create |

**Перед началом работы (всегда):**
```powershell
git pull
```

**После работы:**
```powershell
git add <свои файлы>
git commit -m "feat(server): что сделал"   # для Dev A
git commit -m "feat(client): что сделал"   # для Dev B
git push
```

**Бранчи:**
- `main` — стабильно, всегда работает
- `dev` — текущая работа (по умолчанию пушим сюда)
- `feat/<name>` — большие фичи (потом мержим в `dev` через PR)

# 📝 ИНСТРУКЦИЯ ДЛЯ CLAUDE (читай оба Claude'а у обоих разработчиков)

При работе над этим проектом:

1. **Прежде чем начать любой этап** — спроси разработчика: «Начинаем этап N или отдыхаем?»
2. **Перед написанием кода** — `git pull`, чтоб не было конфликтов с другим разработчиком
3. **Не лезь в зону другого разработчика** без необходимости (см. таблицу зон выше)
4. **После каждой задачи** — коммитить + пушить, чтобы другая сторона могла подтянуть
5. **Источник истины по геймдизайну** — `CLAUDE.md`. Если в `PLAN.md` и `CLAUDE.md` расхождение — `CLAUDE.md` прав
6. **Все цифры баланса** — из `CLAUDE.md`, конкретно из секций «ЭКОНОМИКА» и «REBIRTH»
7. **Тестирование** — после каждого этапа оба разработчика плейтестят вдвоём через Team Create
8. **Если что-то непонятно** — спросить разработчика, а не додумывать
